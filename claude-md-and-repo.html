Your AI's Instruction Manual

CLAUDE.md is a special file Claude reads at the start of every conversation

Include bash commands, code style, workflow rules — persistent context Claude can't infer from code alone

Run /init to generate a starter file, then refine over time

Check it into git so the whole team benefits

Compound engineering: every correction you add makes every future Claude session smarter

What Goes in CLAUDE.md

Include

  • + Bash commands Claude can't guess
  • + Code style rules that differ from defaults
  • + Testing instructions & preferred runners
  • + Repo etiquette (branch naming, PR conventions)
  • + Architectural decisions
  • + Common gotchas

Skip

  • − Anything Claude figures out from code
  • − Standard language conventions
  • − Detailed API docs (link instead)
  • − Info that changes frequently
  • − File-by-file codebase descriptions
  • − Self-evident practices (“write clean code”)

There’s a ~150–200 instruction budget before compliance drops off. Every line should earn its place.

— Anthropic best practices

Give Claude a Way to Verify Its Work

“This is the single highest-leverage thing you can do.”

Before

“Implement email validation”

After

“Write validateEmail. Test cases: user@example.com → true, 'invalid' → false. Run tests after implementing.”

Tests, screenshots, linters, bash commands — invest in making verification rock-solid. This 2–3x the quality.

The Recommended Workflow

1

Explore

Hit Shift+Tab for Plan Mode, read files, understand the codebase

2

Plan

Ask Claude for a detailed implementation plan

3

Implement

Switch to Normal Mode, let Claude code against the plan

4

Commit

Descriptive message, open a PR

Go back and forth with Claude until you like the plan, then switch to auto-accept. Claude usually 1-shots it.

Skip planning for trivial tasks — typos, single-line changes, simple renames.

Teaching Claude New Tricks

Skills extend Claude's knowledge with project-specific workflows. Here's a real one from the Anthropic docs:

# .claude/skills/fix-issue/SKILL.md --- name: fix-issue description: Fix a GitHub issue disable-model-invocation: true --- Analyze and fix the GitHub issue: $ARGUMENTS 1. Use `gh issue view` to get details 2. Search the codebase for relevant files 3. Implement the fix 4. Write and run tests to verify 5. Create a descriptive commit message 6. Push and create a PR

Type /fix-issue 1234 and Claude handles the rest.

Skills can also inject live data with !`gh pr diff` — shell output gets pulled in before Claude sees the prompt.

What a Good Project Looks Like

my-app/ .env.local ← secrets (gitignored) .gitignore ← what Git should skip .mcp.json ← external tools (Slack, Jira, DBs) CLAUDE.md ← AI instructions README.md ← human instructions package.json ← dependencies .claude/ skills/ ← reusable AI workflows commands/ ← slash commands agents/ ← custom subagents settings.json ← permissions, hooks src/ app/ ← pages & routes components/ ← reusable UI lib/ ← utilities & helpers public/ ← static assets

1

Skills = reusable domain knowledge that scales across your team.

3

Verification = the #1 lever for quality. Give Claude a way to check its own work.
