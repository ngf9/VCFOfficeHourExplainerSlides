The Terminal Is the Engine Behind Every Tool

A text-based interface to your computer's operating system. You type commands, the terminal returns output.

Every tool you use — VS Code, Claude Code, npm, git — runs through the terminal under the hood. Learning a handful of commands gives you direct control.

You only need about 10 commands to be productive.

Your Shell Is Just a Program That Reads Commands

These are all shells — the interpreter that runs inside the terminal.

Bash

The original command language. Historically the default on macOS and Linux.

Zsh

A modern shell that replaced Bash as macOS default in 2019. Better defaults, smarter features.

Oh My Zsh

Not a shell — a framework on top of Zsh. Adds themes, plugins, and helpful shortcuts.

Always Know Where You Are

Your file system

~/
  ├── Documents/
  │   └── my-project/ ← you are here
  │       ├── src/
  │       ├── package.json
  │       └── README.md
  └── Desktop/
$ pwd
/Users/you/Documents/my-project
$ cd src
# moved into src/ — use cd .. to go back
$ ls
src/  package.json  README.md

You Can Create and Delete Files Without Leaving the Terminal

$ mkdir my-app
# creates a new folder called my-app
$ touch index.js
# creates an empty file called index.js
$ rm index.js
# deletes the file (no undo!)

rm is permanent. There's no trash can in the terminal. Use rm -r folder-name to delete a folder and its contents.

One Command Opens Your Entire Project

$ code .
# VS Code opens with your project loaded

Opens the current folder in VS Code. The fastest way to get your editor open with the right project loaded.

PRO TIP Make sure you've installed the code command from VS Code's command palette (Cmd+Shift+P → "Install code command")

Install Once, Run Every Day

$ npm install

$ npm run dev

You must run these inside your project folder. Check with ls — if you see package.json, you're in the right place.

Git Commands Follow a Simple Pattern

The commands you'll use every day. See the GitHub 101 deck for the full picture.

git clone <url> Download a repo to your machine
git status See what's changed since last commit
git add . Stage all changes for commit
git commit -m "msg" Save a snapshot with a message
git push Upload your commits to GitHub
git checkout -b name Create and switch to a new branch

Every Error Has a One-Line Fix

command not found

You're in the wrong folder or the tool isn't installed.

EACCES permission denied

Try prefixing with sudo (carefully).

port already in use

Another dev server is running; kill it or use a different port.
